# 도쿄 가족여행 플래너 — 프로젝트 정리 (GitHub 연결 전까지)

새 대화창에서 이어서 작업하실 때, 이 문서를 첨부하고 "이 앱을 GitHub Pages에 배포하고 싶어" 라고 시작하시면 됩니다.

## 개요
- **목적**: 가족 도쿄 3박 4일 여행을 위한 실시간 지도 & AI 연동 일정/경비/아카이빙 웹앱
- **형태**: 단일 HTML 파일. 빌드 과정 없이 브라우저에서 바로 열림 (React/Babel/Tailwind 전부 CDN 로드)
- **완성된 파일명**: `TokyoTripPlanner.html` (= `index.html`, GitHub Pages 배포용으로 이름 맞춰서 동일 내용 별도 제공됨)

## 기술 스택
- React 18 (UMD, cdnjs.cloudflare.com에서 로드 — unpkg는 Claude 아티팩트 환경에서 막혀서 cdnjs로 교체함)
- ReactDOM 18 (UMD)
- Babel Standalone (JSX를 브라우저에서 즉석 변환)
- Tailwind CSS (CDN 버전, JIT 컴파일)
- 아이콘: lucide-react 대신 직접 만든 인라인 SVG 컴포넌트 사용 (외부 아이콘 패키지 의존성 없음)
- Google Maps JavaScript API + Places API (지도 렌더링, 장소 검색/자동완성, 실제 이동거리·시간 계산용 Distance Matrix API)
- Gemini API (REST 직접 호출, 모델명 `gemini-flash-latest` — 특정 버전명은 계속 서비스 종료되는 문제가 있어서 항상 최신 버전을 가리키는 별칭으로 고정함)

## 디자인 히스토리
1. 처음 요청: "민트 글래스모피즘 & 코지 웜 미니멀" — 민트그린/에메랄드/오프화이트 배경, 파스텔 핑크 포인트, 둥근 카드, 반투명 블러 효과
2. 이후 사용자가 레퍼런스 이미지(Fit/Build/Launch 스타일 대시보드)를 주고 "이 느낌으로" 요청 → **전면 재테마**:
   - 배경: 오프화이트 단색(stone-50)
   - 카드: 블러 없는 플랫 화이트 카드 + 얇은 회색 테두리
   - 버튼/활성 상태: 블랙(stone-900)
   - AI 관련 기능(경로 최적화, 추천 등)만 인디고 그라데이션으로 시그니처 포인트
   - 페이지 상단에 은은한 인디고 그라데이션 배너 추가 (레퍼런스의 배경 느낌 재현)
   - 별점만 예외로 골드/앰버 색 유지
3. PC: 좌측 고정 사이드바 + 우측 그리드. 모바일: 하단 탭바. 사이드바는 불투명 배경 처리(그라데이션이 사이드바까지 번져서 가독성 떨어지던 문제 수정함)

## 핵심 기능 (전부 구현 완료)

### 1. 상단 고정 영역
- 출발/도착 공항: 나리타 국제공항(NRT) 좌표 고정
- 고정 숙소: Google Places Autocomplete로 검색·등록, 실제 좌표 확보, 3박4일 모든 일정의 시작/복귀 허브로 사용

### 2. 일정 & 지도 탭 (Day1~4)
- Day1: 공항→숙소→(중간 장소들)→숙소, Day2/3: 숙소→(중간 장소들)→숙소, Day4: 숙소→(중간 장소들)→공항 — 이 허브 로직은 고정
- 장소 검색(Places Autocomplete, 도쿄 중심 반경 45km로 결과 편향시킴)으로 일정에 장소 추가
- 실제 Google Maps 렌더링, 방문 순서대로 번호 마커 + 폴리라인 경로 표시
- **AI 경로 최적화**: Gemini가 방문 순서 + 구간별 이동수단(지하철/도보/택시)만 제안 → 실제 소요시간·거리는 Google Distance Matrix API로 직접 계산해서 표시 (Gemini가 시간을 지어내던 문제를 이렇게 근본적으로 해결함)
- 방문 순서 수동 변경 가능 (▲▼ 버튼으로 순서 스왑)
- Day별 메모(텍스트) 입력란

### 3. 경비 정산 탭
- 환율 설정 (기본 100엔=900원, 수정 가능)
- 결제자: 아빠 / 가령
- **정산 룰 A**: 가족 공통 경비는 아빠 75% (아빠/엄마/오빠 몫) : 가령 25% 부담
- 미정산 항목 기준으로 "누가 누구에게 얼마를 보내야 하는지" 실시간 계산
- 정산완료 체크 시 계산에서 제외
- 지출별 카테고리 + 비고(메모) 필드

### 4. 아카이빙 탭 (맛집/명소 보관함)
- Places 검색으로 등록, 구글 플레이스 사진 자동 첨부
- 별점(1~5), 가족 댓글 스레드(누구나 의견 남기기, 본인 댓글만 삭제 가능)
- "OO이(가) 추가함" 표시로 누가 등록했는지 표시
- **AI 추천**: "라멘 맛집", "분위기 좋은 카페" 같은 조건을 입력하면 Gemini가 도쿄 안에서만(다른 도시 결과 섞이지 않도록 좌표 재검증 추가함) 추천 + 숙소 기준 실제 도보시간을 Distance Matrix로 계산해서 표시
- 길찾기 버튼(구글맵 딥링크), "일정에 추가" 버튼으로 특정 Day에 바로 편입

### 5. 접근 제어 & 가족 공유 저장
- PIN 게이트 (기본값 4065, 코드 상단 `APP_PIN` 상수에서 변경 가능 — 실제 보안이 아니라 가족 외 실수 방지용이라는 점 사용자에게 안내함)
- 이름 선택 게이트 (아빠/엄마/오빠/가령 중 선택, 기기별로 기억됨)
- Claude 아티팩트의 `window.storage` API로 일정/경비/아카이빙 데이터를 공유 저장 — 같은 게시(publish) 링크로 들어온 가족 모두가 같은 데이터를 실시간으로 봄
- **방금 추가한 안전장치**: `window.storage`가 없는 환경(GitHub Pages 등 클로드 바깥)에서는 자동으로 `localStorage`(기기별 저장)로 폴백 → 에러 메시지 없이 최소한 "내 기기에서는" 계속 저장되도록 처리함

## API 키
- Google Maps API Key, Gemini API Key 둘 다 이미 코드 상단 상수(`GOOGLE_MAPS_API_KEY`, `GEMINI_API_KEY`)에 입력되어 있음 — 실제 키 값은 이 문서에 다시 적지 않았으니, 필요하면 HTML 파일 안에서 직접 확인
- Google Maps 키: Maps JavaScript API, Places API, Directions API, Distance Matrix API 4개 활성화됨, 애플리케이션 제한사항은 현재 "없음"으로 설정됨
- ⚠️ 코드에 실제 API 키가 하드코딩되어 있으므로, 이 HTML 파일이나 이 정리 문서를 가족 외 다른 사람과 공유하지 않도록 계속 주의 필요

## 지금까지 고친 주요 버그들
1. Gemini가 이동시간·거리를 추측해서 부정확했던 문제 → Google Distance Matrix로 실측하도록 전면 수정
2. AI 추천 시 도쿄가 아닌 다른 도시(삿포로 등) 결과가 섞이던 버그 → 도쿄 중심 반경 45km 바이어스 추가
3. Gemini가 가끔 깨진 JSON을 반환해 파싱이 실패하던 문제 → 자동 복구(트레일링 콤마 제거 등) + 부분 복구 + 재시도 로직 추가
4. 숙소를 공유 저장소에서 불러왔을 때 입력창에 표시가 안 되던 버그 (uncontrolled input) → ref로 DOM 값 직접 동기화
5. Gemini 모델명이 서비스 종료(deprecated)되어 404 나던 문제 → 항상 최신 버전을 가리키는 `gemini-flash-latest` 별칭으로 교체
6. 아티팩트 배경 그라데이션이 사이드바까지 번져서 텍스트 가독성이 떨어지던 문제 → 사이드바에 불투명 배경 적용

## 이전 대화에서 GitHub Pages 이전에 맞춰 미리 해둔 작업
- `window.storage`가 없는 환경에서 `localStorage`로 자동 폴백하도록 코드 수정 완료 (위 "접근 제어" 섹션 참고)
- GitHub Pages는 기본적으로 `index.html`을 첫 화면으로 인식하므로, 같은 내용의 `index.html` 파일도 별도로 준비해둠
